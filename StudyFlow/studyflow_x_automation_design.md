# 🎮 StudyFlow AI: X (Twitter) 自動投稿システム & コピーライティング設計書

## 1. システム概要と目的
本システムは、Google NotebookLMの講義要約をゲーム感覚の特訓アセットに変換する「StudyFlow AI」の認知拡大・トラフィック誘導を目的とする自動化システムである。
開発者（小島蒼大）の「泥臭い学習の日常（Build in Public）」と「ゲーム要素（ボスバトル勝利、クエスト達成、獲得XP、学習ストリーク、集中テーマ）」を連動させ、リアルタイムかつ高い人間味を持った投稿を自動生成・投稿（またはドラフト保存）し、X上の教育・個人開発・AI関心層をランディングページへと誘導する。

---

## 2. 自動投稿システムのアーキテクチャ設計
既存の `socialintent-api/x_automation/` に実装されている設計思想を応用し、StudyFlowのイベントログと連動する自動投稿プログラムを構築する。

### ① データ連携フロー
1. **クライアント側イベントトリガー (`app.js`)**
   - ユーザーがボスバトルをクリア、デイリークエストを達成、または一定 of 学習ストリークに到達した際に、ローカルストレージに保存されている学習データ（`studyflow_xp`, `studyflow_streak`, `bossDefeated` 等）をバックエンドAPIに送信。
   - 送信先: `POST /api/studyflow/log-event`
2. **データベースへの記録 (`database.py` への拡張)**
   - 受信したユーザーイベント（イベントタイプ、詳細、獲得XP、日時）を SQLite データベース `socialintent.db` に新設する `studyflow_user_logs` テーブルに記録する。
3. **動的画像生成 (`studyflow_x_generator.py`)**
   - データベースから前日〜当日の学習データを抽出し、Pythonの `Pillow` (PIL) または `matplotlib` を用いて、SNSシェア用の「リザルトカード（ボス撃破証明書 / クエスト達成ログ）」をダークネオン風に自動レンダリングする。
   - デザインは、ユーザーが適用している現在の「集中環境テーマ（Amethyst / Emerald / Crimson / Ice）」の色調を自動反映する。
4. **X投稿オートメーション (`studyflow_x_scheduler.py`)**
   - 毎日午後9:00（夜の学習ラップアップ時間）にバックグラウンドタスクが起動。
   - 当日の最良の学習成果（ボス撃破 ＞ クエスト全制覇 ＞ 継続ストリーク）を選択し、生成した画像を添付してX API経由でスレッド投稿。
   - 認証キー未設定時は `C:\COO\メモ\X投稿ドラフト\` にMarkdownとして自動下書き保存する。

### ② データベース追加スキーマ (SQLite)
```sql
CREATE TABLE IF NOT EXISTS studyflow_user_logs (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    event_type TEXT NOT NULL, -- 'boss_victory', 'quest_clear', 'streak_milestone', 'level_up'
    detail_json TEXT NOT NULL, -- ボス名、獲得XP、選択テーマ等のJSON
    logged_at TEXT NOT NULL
);
```

## 3. コピーライティング原則（Obsidian Voice の遵守）
ライティング部門の `obsidian_voice_profile.md` および `copywriting.md` に基づき、AI的な「完璧すぎる宣伝」を徹底的に排除する。

- **MUST（やるべきこと）**
  - **公的Xアカウント向けの口調**: 丁寧語（です・ます調）を使用しつつ、泥臭い本音を混ぜる。
  - **感情のコントラスト**: 序盤で「失敗」「苦戦」を提示し、中盤〜終盤で「それでも這い上がる」姿勢を見せる。
  - **戦略的な絵文字利用**: 箇条書きや重要な数字の先頭に1〜2個程度に絞って配置（😭=弱音、💡=知見、📊=データ、🚀=行動）。
  - **具体的な数値**: 獲得XP数、クリアしたクエスト数、継続日数、HP残量などを細かく記述する。
- **MUST NOT（絶対禁止）**
  - 過度な売り込み（「革命的！」「今すぐ登録！」等のキャッチコピーは厳禁）。
  - ビジネス抽象語（「学習シナジー」「教育エコシステム」など）の使用。
  - 体言止めの多用（述語まで丁寧に書き切る）。
  - 機械的な文章構成（人間がその場で書いたような揺らぎを持たせる）。

## 5. 各部門の連携アクション（Cooperation Guide）

### 📢 マーケティング部門 (marketing_strategist) へのアクション
- Xプロフィール（Bio）に `https://studyflow-ai.netlify.app/` への直接リンクを常時配置し、自動ツイートからの流入を全てランディングページへ誘導する。
- ターゲットである「NotebookLMを活用する試験受験生・ビジネス学習者」に向けて、特定のハッシュタグのインプレッション傾向を監視する。
- Xからのトラフィックを計測するために、UTMパラメータ（`?utm_source=x&utm_medium=auto_daily`）の利用を検討する。

### ✍️ ライティング部門 (copywriter) へのアクション
- 週に一度、自動投稿のインプレッションと「いいね」数をチェックする。
- 反応が良かった投稿文体があれば `obsidian_voice_profile.md` に追記し、悪かった文体（AI臭さが残っていたもの）は `NGパターン` に追加してテンプレートをブラッシュアップする。
